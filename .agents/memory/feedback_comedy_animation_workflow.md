Comedy/character-based animated videos follow the SAME 8-step gated flow as tinh-dao-video skill, NOT a shortcut. Every step gates on user approval.

**Why:** First pilot "Tắt Điện Thoại" (comedy-pilot-01) was rendered ad-hoc — no character lock, no brainstorm, no beat sheet, no script approval. User feedback: "hài như cc chán thế" + called out that I skipped the proper workflow. Skipping leads to bland/aimless output because there's no intentional comedic structure.

**How to apply:** For any animated comedy/character video:

1. **Character lock** — finalize mascot(s) design: body + face + signature prop + color palette + voice assignment. Locked = no changes mid-production.
2. **Concept brainstorm** — present 3 distinct pitches (2 lines each), user picks 1
3. **Beat sheet** — 5-7 beats, each 3-6s, with action + emotion per beat. User approves structure BEFORE dialogue
4. **Script approval** — full dialogue + action + tone notes per beat. User approves line-by-line
5. **Voice casting** — sample 1 line per character, user confirms voice fit
6. **Storyboard mini** — 1 rough frame per beat showing position/pose/bubbles. User reviews layout
7. **TTS + assemble** — render voice, compose Remotion, smoke test frame
8. **Render + review** — full render, preview frames, feedback

**Never skip:** steps 1, 2, 3, 4, 6. Especially character lock before writing script — if character design is still draft you cannot commit to their voice/behavior.

Save this parallel to tinh-dao-video skill. Comedy series is its own skill domain (`comedy-animation/`) — will need its own SKILL.md file eventually.
